1
ตั้งชื่อและระบุหมวด

เริ่มจากชื่อผลิตภัณฑ์ กลุ่มนวัตกรรม กลุ่มการใช้ และสรรพคุณหลักก่อนค่ะ

2
ใส่ข้อมูลผู้ผลิตและรูป

กรอกอำเภอ หน่วยงาน ผู้ผลิต พร้อมแนบรูป 2 รูปเพื่อให้ฐานข้อมูลดูน่าใช้มากขึ้น

3
คำนวณต้นทุนและราคา

กรอกต้นทุนย่อยแล้วกดให้ระบบช่วยคิด ราคาขายจะดูเป็นมืออาชีพขึ้นทันที

4
กดหางานวิจัยและวิเคราะห์

ให้แอปช่วยเติมลิงก์งานวิจัยและเขียนบทวิเคราะห์เบื้องต้น จากนั้นค่อยปรับถ้อยคำได้ค่ะ

สูตรคำนวณราคาขายแนะนำ ต้นทุนรวม = วัตถุดิบ + บรรจุภัณฑ์ + แรงงาน + ค่าใช้จ่ายแฝง + โลจิสติกส์ ราคาขายแนะนำ = ต้นทุนรวม ÷ (1 - อัตรากำไรเป้าหมาย/100) กำไรขั้นต้นต่อชิ้น = ราคาขาย - ต้นทุนรวม

กรอกตัวเลขด้านซ้ายแล้วกดคำนวณ ระบบจะเสนอราคาขายที่เหมาะสม พร้อมอธิบายวิธีคิด

งานวิจัยและแหล่งอ้างอิง

หน้านี้รวบรวมลิงก์แหล่งข้อมูลวิชาการที่เชื่อถือได้ และสามารถใช้ปุ่มค้นหางานวิจัยอัตโนมัติในหน้า “เพิ่มรายการใหม่” หรือหน้า “รายละเอียดผลิตภัณฑ์” เพื่อให้ระบบไปค้นมาลงข้อมูลให้

ตั้งค่าซิงก์ออนไลน์สำหรับใช้งานบน GitHub Pages

อยากให้ทุกคนเห็นข้อมูลชุดเดียวกัน ต้องมีฐานข้อมูลกลางค่ะ

หน้านี้ทำไว้สำหรับเชื่อมต่อฐานข้อมูลกลางแบบง่าย ๆ เมื่อเปิดใช้แล้ว การเพิ่มรายการใหม่ แก้ไข หรือลบข้อมูล จะซิงก์ขึ้น cloud และผู้ใช้คนอื่นที่เปิดเว็บเดียวกันก็จะดึงข้อมูลชุดเดียวกันมาเห็นได้

GitHub Pages = หน้าเว็บ Supabase = ฐานข้อมูลกลาง ทุกคนเห็นข้อมูลเดียวกัน
วาง Project URL ของ Supabase เช่น https://xxxx.supabase.co
ใช้ public anon key เท่านั้น ไม่ใส่ service_role key ลงในไฟล์นี้นะคะ เดี๋ยวจะใจหายเหมือนทำทองหายค่ะ
แนะนำใช้ตาราง app_documents
ใช้ชื่อเดียวกันทุกคนเพื่อแชร์ฐานข้อมูลชุดเดียวกัน
เปิดไว้จะสะดวกที่สุดสำหรับใช้งานจริงในหน่วยงาน
ยังไม่ได้ตั้งค่าซิงก์ออนไลน์

ข้อมูลจะอยู่ในเครื่องนี้ก่อน ยังไม่แชร์กับคนอื่น
ถ้าเอาเว็บขึ้น GitHub Pages โดยไม่ตั้งค่าฐานข้อมูลกลาง ทุกคนจะเห็น “หน้าเว็บเวอร์ชันเดียวกัน” แต่จะไม่เห็น “ข้อมูลที่กรอกใหม่ของกันและกัน” ค่ะ
โครงสร้างตารางที่แนะนำใน Supabase
create table if not exists public.app_documents ( id text primary key, payload jsonb not null default '[]'::jsonb, updated_at timestamptz not null default now() );
เมื่อใช้ตารางนี้ แอปจะเก็บรายการทั้งหมดไว้ใน payload ก้อนเดียว ทำให้เริ่มต้นง่ายและเหมาะกับงานระดับหน่วยงานที่ต้องการใช้งานเร็ว

เกี่ยวกับระบบ

  • เป็นโปรแกรม HTML แบบไฟล์เดียว เปิดใช้งานบน browser ได้ทันที
  • รองรับการเพิ่มรายการใหม่และบันทึกข้อมูลลงเครื่องด้วย Local Storage
  • รองรับรูปผลิตภัณฑ์ 2 รูปต่อรายการ
  • มี Dashboard สรุปแยกตามอำเภอและหน่วยบริการ/หน่วยงาน
  • มีปุ่มค้นหางานวิจัยอัตโนมัติ และปุ่มวิเคราะห์อัตโนมัติพร้อมบันทึกผลไว้ในรายการ
